ℹ️ How Centralized Keys Work
🔑
Admin Sets Key
You enter the Gemini API key here once. It's stored encrypted in Supabase app_settings table.
📡
App Fetches on Load
dashboard.html fetches the key from Supabase on startup. No hardcoded keys in frontend code.
👤
Users Auto-Use It
All authenticated users automatically use the centrally managed key. No setup needed on their end.
⚠️ Required Supabase setup: Run this SQL in your Supabase SQL editor: 
CREATE TABLE IF NOT EXISTS app_settings (
  key text PRIMARY KEY,
  value text,
  updated_at timestamptz DEFAULT now()
);
-- Allow all authenticated users to READ settings
CREATE POLICY "allow_read" ON app_settings FOR SELECT TO authenticated USING (true);
-- Allow only admin (update your admin user ID) to WRITE
CREATE POLICY "allow_admin_write" ON app_settings FOR ALL USING (auth.uid() = 'YOUR_ADMIN_USER_ID');
